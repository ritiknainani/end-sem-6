## LEX Tool — General Theory Questions

### Q1. What is LEX?
**A:** LEX is a tool that automatically generates a **lexical analyzer** (scanner) from a specification file containing regular expressions and associated actions written in C. It produces a C program (`lex.yy.c`).

### Q2. What is the structure of a LEX program?
**A:** Three sections separated by `%%`:
```
%{
  /* Definitions section — C declarations, #include, global variables */
%}
/* Definitions — macros, named regex patterns */
%%
/* Rules section — regex patterns and actions */
pattern1   { action1; }
pattern2   { action2; }
%%
/* User subroutines section — C functions including main() */
```

### Q3. How do you compile and run a LEX program?
**A:**
```bash
lex program.l          # Generates lex.yy.c
cc lex.yy.c -ll        # Compile with LEX library (-lfl on some systems)
./a.out                 # Run
```

### Q4. What is `yytext`?
**A:** `yytext` is a global character pointer that holds the **text of the current token** (the string matched by the current pattern). It's the actual matched lexeme.

### Q5. What is `yyleng`?
**A:** `yyleng` is an integer that holds the **length** of the matched string stored in `yytext`.

### Q6. What is `yylex()`?
**A:** `yylex()` is the main scanner function generated by LEX. It reads input, matches patterns, and executes the corresponding actions. It returns when it encounters `return` in an action or reaches EOF.

### Q7. What is `yyin`?
**A:** `yyin` is a `FILE*` pointer specifying the input source for the lexer. By default it's `stdin`. You can set it to read from a file: `yyin = fopen("input.txt", "r");`

### Q8. What is `yyout`?
**A:** `yyout` is a `FILE*` pointer specifying the output destination. By default it's `stdout`.

### Q9. What is `yywrap()`?
**A:** `yywrap()` is called when the lexer reaches end-of-input. If it returns 1, scanning terminates. If 0, scanning continues (useful for processing multiple files). Default implementation returns 1.

### Q10. What is `ECHO` in LEX?
**A:** `ECHO` is a macro that writes the matched text (`yytext`) to the output. It's equivalent to `fprintf(yyout, "%s", yytext)`. The default action for unmatched input is `ECHO`.

### Q11. What happens to input that doesn't match any rule?
**A:** The default action is `ECHO` — it's printed to the output as-is. This is why unrecognized characters pass through unchanged unless explicitly handled.

### Q12. What is the difference between LEX and YACC?
**A:**
| LEX | YACC |
|-----|------|
| Lexical analyzer generator | Parser generator |
| Uses regular expressions | Uses context-free grammars |
| Generates scanner | Generates parser |
| Produces `lex.yy.c` | Produces `y.tab.c` |
| Handles tokens | Handles syntax structure |

### Q13. What is the longest match rule in LEX?
**A:** When multiple patterns match, LEX chooses the one that matches the **longest string**. If two patterns match the same length, the one listed **first** in the rules section is chosen.

### Q14. What is the difference between LEX and Flex?
**A:** Flex (Fast LEX) is an open-source, faster, and more portable version of LEX. They use the same syntax. Flex generates more optimized scanners and is more widely used today.

### Q15. What regular expression operators does LEX support?
**A:**
| Operator | Meaning |
|----------|---------|
| `.` | Any character except newline |
| `*` | Zero or more |
| `+` | One or more |
| `?` | Zero or one |
| `[]` | Character class |
| `[^]` | Negated character class |
| `\|` | Alternation (OR) |
| `()` | Grouping |
| `^` | Start of line |
| `$` | End of line |
| `{}` | Repetition count or macro expansion |
| `""` | Literal string (no special meaning) |
| `\` | Escape character |

## LEX Program-Specific Viva Questions

### Program 1: Check Vowel or Consonant

**Q: What regex matches vowels?**
A: `[aeiouAEIOU]`

**Q: How do you handle uppercase and lowercase?**
A: Use character class `[aeiouAEIOU]` or convert to lowercase first.

**Q: What about non-alphabetic input?**
A: Add a rule for `[^a-zA-Z]` or `.` to handle digits, spaces, and special characters separately.

---

### Program 2: Check Word or Number

**Q: What regex matches a word? A number?**
A: Word: `[a-zA-Z]+`, Number: `[0-9]+`

**Q: What if input contains both letters and digits like "abc123"?**
A: The longest match rule applies. `[a-zA-Z0-9]+` would match the whole string. Separate rules would match `abc` as word and `123` as number.

**Q: How would you handle floating-point numbers?**
A: Use pattern `[0-9]+\.[0-9]+` for decimals.

---

### Program 3: Count Lines, Words, Characters

**Q: What regex counts a line?**
A: `\n` matches newline. Increment line counter in its action.

**Q: How do you count words?**
A: Pattern `[a-zA-Z]+` or `[^ \t\n]+` matches a word. Increment word counter.

**Q: How do you count characters?**
A: Use `yyleng` — add it to character count for every matched pattern. Or use `.` to match each character.

**Q: What's the difference between `\n` and `\r\n`?**
A: `\n` is Unix line ending. `\r\n` is Windows (CRLF). LEX typically handles `\n` only.

---

### Program 4: Implement Lexical Analyzer (LEX)

**Q: What tokens does your lexer recognize?**
A: Keywords (`if`, `else`, `int`, `float`, etc.), identifiers (`[a-zA-Z_][a-zA-Z0-9_]*`), numbers (`[0-9]+`), operators (`+`, `-`, `*`, `/`, `=`, etc.), and delimiters (`;`, `{`, `}`, `(`, `)`).

**Q: How does LEX handle multi-character operators like `<=` or `==`?**
A: Define explicit patterns for them. Due to longest match rule, `<=` is matched before `<`.

**Q: What is the order of rules importance?**
A: Keywords must be listed **before** the identifier pattern, otherwise keywords would be matched as identifiers (since `int` also matches `[a-zA-Z]+`).

---

### Program 5: Check Digit or Alphabet

**Q: What regex matches a single digit?**
A: `[0-9]`

**Q: What regex matches a single alphabet?**
A: `[a-zA-Z]`

**Q: How would you handle multi-character input?**
A: Use `[a-zA-Z]+` for words and `[0-9]+` for numbers to group consecutive characters.

---

### Program 6: Count Vowels in a String

**Q: What regex matches vowels?**
A: `[aeiouAEIOU]`

**Q: How do you count them?**
A: Use a global counter variable. In the action for the vowel pattern, increment the counter. Print at end.

**Q: Do you count uppercase and lowercase separately or together?**
A: Depends on requirement. Typically together using `[aeiouAEIOU]`.

---

### Program 7: Convert Lowercase to Uppercase

**Q: What regex matches lowercase letters?**
A: `[a-z]`

**Q: How do you convert?**
A: `printf("%c", yytext[0] - 32);` or `printf("%c", toupper(yytext[0]));`

**Q: Why subtract 32?**
A: In ASCII, the difference between lowercase and uppercase letters is 32. E.g., 'a' = 97, 'A' = 65, difference = 32.

---

### Program 8: Count Spaces and Tabs

**Q: What regex matches space? Tab?**
A: Space: `" "` or `[ ]`. Tab: `\t`.

**Q: Can you match both in one pattern?**
A: Yes, `[ \t]` matches either space or tab.

**Q: How do you differentiate between leading and embedded whitespace?**
A: You'd need state tracking or context-specific patterns. Basic LEX counts all occurrences.

---

### Program 9: Identify Special Characters

**Q: What are special characters?**
A: Characters that are not letters, digits, or whitespace. E.g., `!`, `@`, `#`, `$`, `%`, `^`, `&`, `*`, etc.

**Q: What regex matches special characters?**
A: `[^a-zA-Z0-9 \t\n]` — negated character class excluding alphanumeric and whitespace.

**Q: How do you escape special characters in LEX regex?**
A: Use `\` before the character. E.g., `\*`, `\.`, `\+`. Or put them in quotes: `"*"`.

---

### Program 10: Count Digits, Alphabets, Special Symbols Separately

**Q: How do you maintain three separate counts?**
A: Three global variables: `digit_count`, `alpha_count`, `special_count`. Each pattern's action increments the appropriate counter.

**Q: What patterns do you use?**
A:
```
[0-9]    { digit_count++; }
[a-zA-Z] { alpha_count++; }
[^ \t\n] { special_count++; }
```
Note: order matters — digits and alphabets must come before the catch-all.

---

### Program 11: Check Valid Identifier

**Q: What is a valid identifier?**
A: Starts with a letter or underscore, followed by zero or more letters, digits, or underscores. Cannot be a keyword.

**Q: What regex matches a valid identifier?**
A: `[a-zA-Z_][a-zA-Z0-9_]*`

**Q: What makes an identifier invalid?**
A: Starting with a digit (e.g., `2name`), containing special characters (e.g., `my-var`), or being a keyword.

**Q: How do you check it's not a keyword?**
A: List keyword patterns before the identifier pattern, or check `yytext` against a keyword list.

---

### Program 12: Recognize Keywords

**Q: How do you match keywords?**
A: Explicit patterns for each keyword:
```
"if"      { printf("Keyword: IF\n"); }
"else"    { printf("Keyword: ELSE\n"); }
"int"     { printf("Keyword: INT\n"); }
```

**Q: Why put keywords before identifiers in rules?**
A: If the identifier rule `[a-zA-Z]+` comes first, it would match `if` as an identifier. Keywords must have higher priority (listed first).

**Q: What are the C keywords?**
A: `auto, break, case, char, const, continue, default, do, double, else, enum, extern, float, for, goto, if, int, long, register, return, short, signed, sizeof, static, struct, switch, typedef, union, unsigned, void, volatile, while` (32 keywords in C89).

---

### Program 13: Count Operators and Operands

**Q: What are operators and operands?**
A: **Operators:** `+`, `-`, `*`, `/`, `=`, `<`, `>`, `<=`, `>=`, `==`, `!=`, etc. **Operands:** identifiers and constants (numbers).

**Q: What regex matches operators?**
A: `[+\-*/=<>!%&|^]+` or individual patterns for each operator.

**Q: What is the difference between unary and binary operators?**
A: **Unary** operates on one operand (`-x`, `!flag`). **Binary** operates on two (`a + b`). Context determines which, but LEX only does tokenization, not context analysis.

---

### Program 14: Detect Palindrome Strings

**Q: Can LEX alone detect palindromes?**
A: Not with pure regex (palindrome is not a regular language for arbitrary length). We use C code in the action to reverse and compare the string.

**Q: How do you implement it?**
A: Match a word `[a-zA-Z]+`, then in the action, use C code to reverse `yytext` and compare with the original.

**Q: Why can't regular expressions detect palindromes?**
A: Palindromes of arbitrary length require memory of what was seen before (like a stack), which goes beyond the capability of finite automata / regular languages. Palindromes are a context-free language.

---

### Program 15: Recognize Integer and Floating-Point Numbers

**Q: What regex matches an integer?**
A: `[0-9]+`

**Q: What regex matches a floating-point number?**
A: `[0-9]+\.[0-9]+` or `[0-9]*\.[0-9]+` (to handle `.5`).

**Q: How do you handle negative numbers?**
A: Add an optional sign: `[-+]?[0-9]+` for integers, `[-+]?[0-9]*\.[0-9]+` for floats.

**Q: What about scientific notation like 1.5e10?**
A: `[0-9]+\.?[0-9]*[eE][-+]?[0-9]+`

**Q: How do you handle order of rules for integers and floats?**
A: Put the float pattern **before** the integer pattern, so `3.14` is matched as float, not `3` as integer.

---

### Program 16: Recognize Even or Odd Number

**Q: How do you determine even or odd in LEX?**
A: Match the number `[0-9]+`, then in the action use C code: `if (atoi(yytext) % 2 == 0)` → even, else → odd.

**Q: Can you do it purely with regex?**
A: Yes, for the last digit: `[0-9]*[02468]` for even, `[0-9]*[13579]` for odd. This avoids C code computation.

**Q: What function converts yytext to integer?**
A: `atoi(yytext)` converts string to integer. For long integers, use `atol()` or `strtol()`.

## Bonus: Cross-Cutting Viva Questions (Full Syllabus)

### Q: What are the phases of a compiler?
**A:** 1) Lexical Analysis 2) Syntax Analysis 3) Semantic Analysis 4) Intermediate Code Generation 5) Code Optimization 6) Code Generation. Plus: Symbol Table Manager and Error Handler throughout.

### Q: What is the difference between a compiler and an interpreter?
**A:** A compiler translates the entire program to machine code before execution. An interpreter translates and executes line by line. Compiler is faster at runtime; interpreter gives better error messages.

### Q: What is the difference between a compiler and an assembler?
**A:** A compiler translates high-level language to machine/intermediate code. An assembler translates assembly language (mnemonics) to machine code. Assembler is simpler — nearly 1:1 mapping.

### Q: What is a context-free grammar (CFG)?
**A:** A grammar where each production rule has a single non-terminal on the LHS. Formally: G = (V, T, P, S) where V = non-terminals, T = terminals, P = productions, S = start symbol.

### Q: What is the difference between top-down and bottom-up parsing?
**A:** Top-down starts from the start symbol and derives the input (LL parsers). Bottom-up starts from the input and reduces to the start symbol (LR parsers).

### Q: What is ambiguous grammar?
**A:** A grammar where a string can have **more than one parse tree** (or leftmost/rightmost derivation). E.g., `E → E + E | E * E | id` is ambiguous for `id + id * id`.

### Q: What is a parse tree vs syntax tree (AST)?
**A:** A parse tree shows the complete derivation including all grammar symbols. An AST (Abstract Syntax Tree) is a compressed version that omits redundant nodes (like parentheses and chain productions), showing only the essential structure.

### Q: What are synthesized vs inherited attributes?
**A:** **Synthesized:** Computed from children nodes (bottom-up). **Inherited:** Computed from parent/sibling nodes (top-down). S-attributed grammars have only synthesized attributes.

### Q: What is an assembler? Explain two-pass assembler.
**A:** An assembler converts assembly language to machine code. **Pass 1:** Builds the symbol table (assigns addresses to labels). **Pass 2:** Generates machine code using the symbol table and opcode table.

### Q: What is a linker and loader?
**A:** **Linker:** Combines multiple object files into one executable, resolving external references. **Loader:** Loads the executable into memory and starts execution. Types: absolute loader, relocating loader, dynamic loader.
